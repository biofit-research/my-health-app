# 바이오핏 처방약 매칭 도우미

> 타 병원 처방약을 본원 보유 약품과 자동 매칭하는 단일 HTML 웹앱
> \*Korean prescription drug matching tool — single-file, browser-only, no server required.\*

병원 약제부·외래 진료에서 환자가 가져온 타 병원 처방전을 본원 약품 마스터와 대조해 **동일 약 / 대체 가능 / 처방 불가**를 즉시 판별합니다. 한글 약 이름만 입력해도 HIRA 약제급여목록표를 통해 영문 성분으로 자동 변환되며, 처방전 사진 OCR과 약 모양 식별 외부 링크도 함께 제공합니다.

\---

## 주요 기능

* **자동 매칭** — 한글 약 이름 → 본원 동일 상품명 ✓ / 동일 성분·함량 대체약 🔄 / 미보유 ✗
* **HIRA 통합** — 건강보험심사평가원 약제급여목록표(약 14,000 브랜드) 매핑으로 영문 성분 자동 변환
* **복합제 분석** — 텔로듀오·엠디비카 등 복합제는 각 성분별로 본원 보유 여부 분리 분석
* **함량 단위 자동 변환** — 0.25mg ↔ 250㎍, 0.1g ↔ 100mg 동일 처리
* **제형 무시 매칭** — 정/캡슐/시럽/패치 등 제형 명사 자동 분리 (`올멕정` = `올멕` = `올멕 20mg`)
* **OCR 입력** — Tesseract.js로 처방전 사진 한글 텍스트 추출 + 사전 기반 자동 보정
* **Fuzzy 매칭** — Levenshtein 거리 기반 오타 자동 교정 (`센리카` → `센리카정`)
* **약 모양 식별** — 식약처 의약품안전나라·약학정보원 외부 링크
* **본원 약품 검색** — 상품명·성분명(영문)으로 직접 검색
* **완전 클라이언트 사이드** — 모든 데이터는 브라우저 localStorage에만 저장, 외부 전송 없음

## 사용법

### 1\. 본원 약품 마스터 업로드 (선택)

엑셀 파일을 끌어놓거나 클릭해서 업로드하면 컬럼 매핑 화면이 표시됩니다. 한글 약품명·성분명 컬럼을 지정하면 됩니다. 재고/처방구분 컬럼이 있으면 입원·외래·품절 분류도 자동 처리됩니다.

> 마스터 없이도 HIRA로 약 이름과 성분 확인은 가능합니다.

### 2\. HIRA 약제급여목록표 업로드 (선택, 강력 권장)

설정(⚙) → HIRA 매핑 탭 → \[HIRA 사이트 열기] → 최신 게시물의 첨부 엑셀 다운로드 → 받은 .xls를 끌어놓기.

이 단계가 있으면 **한글 약 이름만 입력해도 영문 성분으로 자동 변환**됩니다.

### 3\. 처방 입력 → 매칭

```
노바스크정 5mg
센리카 75mg
리피토 10mg
듀로제식디트랜스 25mcg/h
텔로듀오 80/5mg

# 정확한 매칭이 필요하면 성분 직접 입력
자나팜정 0.25mg | alprazolam | 0.25mg
```

`Ctrl/Cmd + Enter`로 매칭 시작.

## 입력 형식

가장 단순하게는 **한 줄에 한 약 이름**만 적어도 됩니다.

|형식|예시|
|-|-|
|약 이름만|`노바스크정 5mg`|
|약 이름 (제형 생략)|`노바스크 5mg` 또는 `노바스크`|
|정확한 성분 지정|`자나팜정 0.25mg \| alprazolam \| 0.25mg`|
|주석 라인|`# 이 줄은 무시됨` (`#` 또는 `//`로 시작)|

### 인식되는 함량 단위

`mg`, `mcg`, `μg`, `㎍`, `ug`, `g`, `iu`, `ml`, `cc`, `%`, `mg/g`, `mg/ml`, `mEq`

### 슬래시 함량 (복합제)

`텔로듀오 80/5mg`, `엠디비카 320/12.5mg` 같은 복합제 표기 지원.

### 주의 — 일반명 한글은 인식 불가

`디곡신`·`메토포민`·`심바스타틴` 같은 **일반명(generic) 한글**은 HIRA 등재 브랜드명이 아니므로 자동 인식되지 않습니다. 처방전의 정확한 **제품명**을 입력하거나 영문 성분으로 직접 검색하세요.

## 설치 / 배포

### 로컬 사용

HTML 파일을 다운받아 브라우저에서 열기만 하면 끝. 별도 서버·빌드 도구 불필요.

```bash
git clone https://biofit-research.github.io/my-biofit.git
cd <리포지토리명>
# 브라우저로 prescription-matcher 열기
open biofit\_v3.html        # macOS
xdg-open biofit\_v3.html    # Linux
start biofit\_v3.html       # Windows
```

### GitHub Pages 배포

1. 리포지토리 Settings → Pages
2. Source: Deploy from a branch (main / root)
3. `https://biofit-research.github.io/my-biofit/` 로 접속

### 사내 서버 배포

`drug-matcher`을 정적 파일 서버(Nginx, Apache, Caddy 등)에 그대로 올리면 됩니다. 인터넷 연결은 OCR 라이브러리(Tesseract.js)와 SheetJS CDN 로딩에만 사용되므로, 폐쇄망에서는 두 라이브러리를 로컬에 함께 호스팅하면 됩니다.

## 기술 스택

* **순수 HTML + CSS + Vanilla JavaScript** — 빌드·번들·프레임워크 없음
* [**Tesseract.js**](https://github.com/naptha/tesseract.js) **v5** — 한국어 OCR
* [**SheetJS**](https://sheetjs.com/) **(xlsx) v0.18.5** — 엑셀 파싱
* **localStorage** — 클라이언트 측 데이터 저장 (마스터 + HIRA 매핑 + 브랜딩 설정)
* **자체 구현** — Levenshtein 거리 기반 fuzzy 매칭, 한글 제형 정규화, 함량 단위 변환

\---

## 데이터 출처

* **본원 약품 마스터** — 사용자가 직접 업로드한 엑셀 (외부 노출 없음)
* **약제급여목록표** — 건강보험심사평가원(HIRA) 공식 게시 자료 ([링크](https://www.hira.or.kr/bbsDummy.do?pgmid=HIRAA030014050000))
* **약 모양 식별** — 식약처 의약품안전나라, 약학정보원 (외부 링크 연결만)

## 프라이버시 / 보안

* 모든 데이터는 **사용자의 브라우저 localStorage에만** 저장됩니다
* 외부 서버 전송 없음 (네트워크 요청은 라이브러리 CDN 로딩이 전부)
* 환자 정보 입력 권장하지 않음 — 약 이름만 입력해 매칭에 사용
* 매핑 데이터는 \[마스터 데이터 삭제] 버튼으로 즉시 제거 가능
